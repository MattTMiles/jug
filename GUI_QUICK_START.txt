                    JUG GUI - Quick Start Guide

LAUNCH:
-------
$ jug-gui                                    # Empty GUI
$ jug-gui pulsar.par pulsar.tim             # Load files directly
$ jug-gui --gpu pulsar.par pulsar.tim       # Load with GPU mode
$ jug-gui --fit F2 F3 pulsar.par pulsar.tim # Add parameters not in .par
$ jug-gui --help                            # Show help

Example with J1909-3744:
$ jug-gui data/pulsars/J1909-3744_tdb.par data/pulsars/J1909-3744.tim


WORKFLOW:
---------
1. File ‚Üí Open .par...     (select your .par file)
2. File ‚Üí Open .tim...     (select your .tim file)
   ‚Üí Prefit residuals automatically plotted

3. Select parameters to fit:
   ‚òë F0, F1 (default)
   ‚òê F2, DM, DM1, DM2 (optional)

4. Click "Run Fit" button
   ‚Üí Wait 0.8-2 seconds (first fit)
   ‚Üí Subsequent fits: ~0.01 seconds!
   ‚Üí Fit results dialog appears

5. View statistics in right panel:
   - RMS (microseconds)
   - Iterations
   - TOAs
   - œá¬≤/dof

6. Delete TOAs: Click+drag to draw box, release to delete
   - Press 'U' to undo last deletion
   - RMS automatically updates

7. Click "Reset to Prefit" to restore original residuals


KEYBOARD SHORTCUTS:
-------------------
Ctrl+P  - Open .par file
Ctrl+T  - Open .tim file
Ctrl+F  - Run fit
Ctrl+R  - Reset to prefit
Ctrl+0  - Zoom to fit plot
U       - Undo last TOA deletion
Ctrl+Q  - Quit

PERFORMANCE:
------------
JUG is heavily optimized:
- First fit: ~0.8 seconds (caching setup)
- Subsequent fits: ~0.01 seconds (257x faster!)
- Warm session: ~162ms (4.5x faster with geometry cache)

To prefetch data for offline use:
$ python -m jug.scripts.download_data

To check cache status:
$ python -m jug.scripts.download_data --status


TROUBLESHOOTING:
----------------
- GUI won't launch?
  ‚Üí Use conda installation (see INSTALL.md)
  ‚Üí Run: mamba env create -f environment.yml

- Fit fails?
  ‚Üí Check .par and .tim files are valid
  ‚Üí Try selecting different parameters

- GUI freezes?
  ‚Üí Should NOT happen! Fitting runs in background thread
  ‚Üí If it does, report as bug
